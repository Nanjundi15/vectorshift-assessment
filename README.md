## вңҰ &nbsp; Overview

**VectorShift Pipeline Builder** is a full-stack, interactive drag-and-drop pipeline editor built as part of the VectorShift Frontend Technical Assessment. It demonstrates a clean component abstraction system, real-time UI logic, polished design, and seamless backend integration вҖ” all wired together into a single, deployable application.

## вңҰ &nbsp; API Reference

<br/>

### `GET /`
Health check вҖ” returns `{ "Ping": "Pong" }`

### `POST /pipelines/parse`

Analyzes the submitted pipeline.

**Request body:**
```json
{
  "nodes": [ { "id": "customInput-1" }, { "id": "llm-1" } ],
  "edges": [ { "source": "customInput-1", "target": "llm-1" } ]
}
```

**Response:**
```json
{
  "num_nodes": 2,
  "num_edges": 1,
  "is_dag": true
}
```

<br/>

| Field | Type | Description |
|-------|------|-------------|
| `num_nodes` | `int` | Total node count in the pipeline |
| `num_edges` | `int` | Total edge count in the pipeline |
| `is_dag` | `bool` | `true` if no cycles detected, `false` otherwise |

## вңҰ &nbsp; DAG Detection вҖ” How it Works

The backend uses an **iterative depth-first search** with three-color marking to detect cycles:

```
WHITE  вҶ’  Node not yet visited
GRAY   вҶ’  Node currently in the DFS stack (being explored)
BLACK  вҶ’  Node fully processed
```

If a `GRAY` node is encountered during traversal, a **back edge** (cycle) has been found вҶ’ `is_dag: false`.

This correctly handles:

- вң… &nbsp; Linear chains &nbsp;`(A вҶ’ B вҶ’ C)`
- вң… &nbsp; Diamond graphs &nbsp;`(A вҶ’ B, A вҶ’ C, B вҶ’ D, C вҶ’ D)`
- вң… &nbsp; Disconnected components &nbsp;`(A вҶ’ B) &nbsp; (C вҶ’ D)`
- вң… &nbsp; Empty pipelines
- вқҢ &nbsp; Self-loops &nbsp;`(A вҶ’ A)`
- вқҢ &nbsp; Two-node cycles &nbsp;`(A вҶ’ B вҶ’ A)`
- вқҢ &nbsp; Large cycles &nbsp;`(A вҶ’ B вҶ’ C вҶ’ A)`
